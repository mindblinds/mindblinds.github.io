github pagesでなんか書こう

githubってなんかプログラマーが色々ゴチョゴチョしてる場所ってイメージだったし、実際そうっぽいんだけど、github.ioというアドレスでwebを公開しているのみつけて、これは便利そうだなー使いたいなーioってなんかかっこいいなーとつねづね思ってた。

でもgithubってプログラマーがゴチョゴチョしてる場所だけあってあんまりエンドユーザーが使いやすいようには作られておらず、どうやってgithub.ioでページ作るのかわからなかった。

それからしばらくgithubのこと忘れてたんだけど、今になってページを作る方法をネットで調べてみたら、なんか書いてることがわかるような気がしてきたので、その通りにやったら、この通りgithubにページ作ることができた。

静的ファイルを置くっていうところをギリギリクリアしただけだし、ちょっと手順から外れたことして壊したらもとに戻せないと思う。その辺は不安だけど、とにかくgithubでページ公開できてやったー！って思った。

一般的なgithub pagesの使い方というのは、githubで開発してきた何かのポートフォリオページをみんなに見てもらうことだと思う。ひろく一般に自分が何を作ってこれはどういうものでこういうことができるしみんな触ってみ？とアテンションを集める。

ぼくは別にそういうのじゃないから、だれも読みたくない日記くらいしか書くことがない。

日記かくのにふつうはブログを使うし、ブログの方が便利だと思う。わざわざ手作業で静的ページをこさえてホスティングサービスで公開するのは時代錯誤な行動だなーと思われることでしょう。

まあそれはそうなんだけど、ブログってなんかソーシャルとかアフィとかレスポンスバトルとか、みんなでワイワイやる場って感じがして、なんかそういうの鬱陶しいなーって思うし、あんまり合わない感じがする。だからぼくははてなでぼっちだしTwitterでもぼっちなんだ。

Tumblrにはかわいい画像を黙々とリブログするという点でのみ連帯する個別的自我の複合体が存在しており、ぼくもそれなんだけど、あそこは黙々とリブログするだけが唯一のコミュニケーションで、とても居心地がよい。

そういう意味で日記も、ここを使って、誰も見てないけど誰でも見れる状態に置いておくという感じにしていきたい。

デザインについて

今のトレンドであるフラットやミニマリズムを意識して作ってる。デザインに詳しいわけではないので、たぶん間違ってるような気がするけど、それっぽい感じは狙っていきたい。あと、今はPCよりスマホでネットする人が多いらしいから、スマホでちゃんと見えるようなおもてなしにしている。chromeでしかチェックしてないけど、iPhoneでちゃんと見られたらいいなーと祈ってる。
